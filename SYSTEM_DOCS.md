# 智能全渠道CRM与AI自动化工作流系统

本文档提供了系统的详细使用说明以及整体架构设计。该系统基于现代化的全栈技术构建，并深度融合了 Google Gemini 语言模型，提供多渠道客情维护、销售报价、基于意图的AI消息生成、以及代理（Agent）自动化调度等功能。

---

## 一、 系统架构与技术栈 (System Architecture)

### 1. 前端层 (Frontend)
- **核心框架**: React 19 + Vite 构建的单页应用 (SPA)。
- **路由管理**: 采用 `react-router-dom` 实现前端无刷新路由跳转。
- **UI与样式**: 基于 `Tailwind CSS` 与 `lucide-react` 图标库构建现代化且具备良好交互的响应式页面设计。
- **动画呈现**: 引入了 `motion` 实现流畅的页面加载与元素交互过渡效果。
- **国际化 (i18n)**: 提供了多语言支持环境设定，通过 Context API 管理。

### 2. 后端层 (Backend)
- **服务器框架**: Node.js + Express.js 构建的 API 服务（位于 `server.ts`）。
- **进程管理**: 运行态时通过 `vite` 挂载中间件提供开发者环境支持，生产阶段通过 `esbuild` 打包为 `server.cjs` 执行。
- **核心路由**:
  - `/api/ai/draft-reply`：依托客户意图及历史对话，利用AI能力自动起草符合客户首选语言习惯的回复信息。
  - `/api/ai/draft-proposal`：基于客户洞察，使用AI动态生成业务提案。
  - `/api/ai/trigger-agent`：触发后台 AI 代理，根据系统预设的系统操作语言汇报自动化执行流程。
  - `/api/ai/vectorize-doc`：支撑知识库文本向量化，便于后续基于 RAG(检索增强生成) 提供智能搜索。

### 3. 数据层与 AI (Data & AI Layer)
- **云AI平台**: 通过官方 `@google/genai` 接入 Google Gemini 语言模型，完成意图识别、自动化提案、摘要、信息润色等。
- **本地存储与模拟持久层**: 
  - `src/services/db.ts`：处理应用级核心实体，利用浏览器缓存 (localStorage) 进行了快速原型化演示存储系统，部分敏感模块（如向量数据库）预留了 `pg+pgvector` PostgreSQL 拓展支持。
  - **核心模型**：`Customer` (客户资料), `InboxMessage` (消息), `Agent` (智能代理), `Quote/Product` (销售与产品), `Media` (媒资) 等。

## 二、 业务设计规范

- **多语言（国际化）生成策略**：
  - **Outbound（对外输出）内容**：如起草给客户的邮件、WhatsApp消息、提案等，**始终以客户在 CRM 中设置的“首选语言 (Preferred Language)”为主**。若客户未设置，则默认退回英文 (English)。
  - **Inbound/内部协作内容**：如 AI Agent 生成的执行日志 (Logs) 、系统的执行状态等内部用户可见范围的信息，**均以系统当前设置的预选语言 (System Language) 为准**。所有确认删除类的弹出框 (Confirm Modal) 都有极佳的用户容错交互。

## 三、 功能模块与使用说明 (User Guide)

系统主导航位于左侧边界，提供以下核心模块的操作入口：

### 1. 仪表盘 (Dashboard)
- **功能概述**：系统全局数据一览，提供关键销售指标更新、当日代办代理任务以及即将产生的收入预估。
- **使用建议**：作为日常工作视窗，可快速跳转到近期的紧急事务处理。

### 2. 收件箱 (Inbox)
- **功能概述**：整合 WhatsApp 与 Email 通讯渠道的统一消息收口中心。
- **使用说明**：
  - 阅读客户消息后，点击右上角的 **“✨ 生成内容 (AI Draft)”** 按钮，AI 会根据识别到的客户语意、意图及客户的 **首选语言** 来自动起草最合适的沟通内容。
  - 能够直接对会话追加内部评论（仅内部同事可见）。

### 3. 客户管理 (Customers/CRM)
- **功能概述**：维护客户联系人，从潜在线索 (Lead) 跟踪至成单转化。
- **使用说明**：
  - **新建/编辑**：能够配置客户的基本信息、联系方式，及重要的 **Preferred Language（偏好沟通语言）**。
  - **客户详情**：在查看客户详情卡片时，能够直观看到带有智能建议的意图分析。可以点击 **“生成方案草稿 (Generate Proposal Draft)”**，AI 将按客户常用语言输出一份经过折扣测算的优化方案。

### 4. 销售 (Sales)
- **功能概述**：集成了产品库及对外报价单 (Quotation) 两大维度。
- **使用说明**：支持关联客户来签发包含指定关联产品项的 Quote 单，一站掌握每笔业务的价格明细，并可以通过 Confirm Modal 安全删除作废的报价。

### 5. 智能代理 (AI Agent Center)
- **功能概述**：定义能够自动分析邮箱、识别高潜线索的驻留机器人数智人中心。
- **使用说明**：配置并查看特定智能机器人日志。点击“Trigger/Test”运行工作流时，后台大语言模型将生成步骤日志，此执行**日志语言与当前的操作员系统语言保持绝对一致**。

### 6. 知识库 (Knowledge Base)
- **功能概述**：系统内嵌的企业电子档案与向量检索库。
- **使用说明**：上传产品PDF、公司介绍等文件即可通过 AI 完成切片向量化 (`Vectorizing`)。在此上传的背景资料未来可直接赋能客服 AI 问答及提案生成。

### 7. 媒体资产 (Media Library)
- **功能概述**：集中的公共图床与附件库管理。
- **使用说明**：
  - 将多张图片一并拖拽进上传弹出窗。支持查看所有资产的空间占比情况。
  - 所有删除均经过二次用户提示校验，防止多媒体素材被误删。

### 8. 用户及系统设置 (Users & Settings)
- **系统用户 (User Management)**：增删子账号（具备误操作规避的防误删二次确认）。
- **设置 (Settings)**：包含外观主题 (Dark/Light mode) 以及**全局系统语言** (Language: zh/en 等等)。切换此处将直接改变系统的显示语言，及所有内部 AI 任务生成和界面文本。
